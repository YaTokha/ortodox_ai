# Функциональные требования (ФТ)

## FR-01 Импорт корпуса
Система должна читать `.txt`-файлы из `data/raw/bible`, `data/raw/commentaries`, `data/raw/sermons` и формировать единый `jsonl`-корпус.

## FR-02 Предобработка
Система должна очищать текст, нормализовывать пробелы и делить длинные документы на фрагменты.

## FR-03 Формирование датасетов
Система должна автоматически формировать train/valid/test выборки.

## FR-04 Дообучение модели
Система должна запускать дообучение GPT-2/ruGPT с LoRA через `train/train_lora.py`.

## FR-05 Оценка качества
Система должна рассчитывать `eval_loss` и `perplexity` на тестовой выборке.

## FR-06 Анализ текста Писания
По API `/api/analyze` система должна выдавать анализ, ключевые темы и релевантные источники.

## FR-07 Генерация проповеди
По API `/api/generate` система должна генерировать черновик проповеди и краткий план.

## FR-08 Retrieval и цитаты
Система должна возвращать список релевантных источников из корпуса с фрагментами и идентификаторами.

## FR-09 Веб-интерфейс
Система должна предоставлять форму для анализа и генерации в браузере.

## FR-10 Проверка сервиса
Система должна иметь endpoint `/api/health` со статусом загрузки модели и адаптера.
